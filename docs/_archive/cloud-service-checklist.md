# 云托管服务确认清单

> 双机 active-active 部署前，需要和云托管方确认的问题。

---

## Redis

1. **实例是否与其他系统共用？**
   如果共用，我们需要给所有 key 加命名空间前缀，避免不同系统之间 key 冲突

2. **主从切换的时间窗口是多少？**
   我们用 Redis 做分布式锁（lease），锁的过期时间和心跳间隔需要根据切换窗口来设计——切换期间锁不能误过期

3. **主从切换时，已有的 Pub/Sub 连接和 XREAD 长连接会怎样？**
   是自动重连还是直接断开？我们需要据此决定业务层的重连策略

4. **主从切换后，Lua script cache 是否会清空？**
   我们用 Lua 脚本做原子操作，如果切换后 cache 清空，需要做自动重载

5. **是否有禁用或限制的命令？**
   特别关注：`KEYS`、`SCRIPT LOAD`、`EVALSHA`、`DEBUG`

6. **最大连接数限制是多少？**
   双机部署，每台机器有连接池，需要确认总连接数不超限

7. **内存上限和淘汰策略是什么？**
   我们用 Redis Stream 缓冲事件数据，需要预估内存用量

---

## TDSQL

1. **兼容的 PostgreSQL / MySQL 版本是多少？**
   影响我们的数据库驱动和 ORM 方言选择

2. **主从切换的时间窗口是多少？**
   影响数据库连接的重试策略配置

3. **最大连接数限制是多少？**
   同 Redis，双机 × 并发数，需要确认不超限

4. **默认的事务隔离级别是什么？**
   我们的并发写入行为依赖隔离级别

5. **是否支持 `SELECT 1` 做连通性检查？**
   健康检查端点需要用
